# 顺序表

## 优点：  

### 随机访问效率高：    
顺序表的元素在内存中是***连续存储***的，因此可以通过***下标***直接访问任意位置的元素，时间复杂度为`O(1)`，这在查找和更新操作中非常高效。  
***存储密度高：*** 顺序表除了存储元素本身所占的空间外，几乎不需要额外的存储空间来存储指针等辅助信息，因此存储密度较高，节省了空间。  
适合元素较少或大小基本固定的情况： 对于元素数量不会频繁变化或者大小基本固定的情况，顺序表的管理和操作比较简单，效率较高。  
无需额外的指针空间： 相比链表等动态分配内存的数据结构，顺序表不需要额外的指针空间来维护元素之间的关系，因此节省了空间。   


## 缺点：

**插入和删除效率低：** 在顺序表中，如果需要在中间或开头插入或删除元素，需要移动大量元素，时间复杂度为`O(n)`，效率较低。
**空间浪费：** 顺序表需要预先分配一定大小的存储空间，如果元素数量不足时会造成部分空间的浪费；如果元素数量超过预分配的大小，需要进行扩容操作，可能会导致大量空间的浪费。  
**容量固定：** 在静态顺序表中，一旦初始化时分配的空间用完，就无法再添加新的元素，除非重新申请更大的空间并将原有数据复制过去。     
操作复杂度不稳定： 在插入或删除操作频繁的情况下，可能会导致大量元素的移动，操作复杂度不稳定。    
综上所述，顺序表适合元素数量相对稳定、需要频繁随机访问元素的场景，但在插入和删除元素频繁、元素数量动态变化较大的情况下，其效率和空间利用率可能不如链表等动态数据结构。
