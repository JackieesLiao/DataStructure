# 队列

##### 顺序队列（`Sequential Queue`）和链表队列（`Linked Queue`）是两种常见的队列实现方式，它们在数据结构和实现方式上有一些区别。   
 
## 顺序队列（`Sequential Queue`） 

顺序队列是使用数组实现的队列，其特点是：

1.使用***数组**来保存队列中的元素。  
2.需要预先指定*队列的最大容量（固定大小）*，即队列长度***不能动态增长***。   
3.需要两个指针 `front` 和 `rear`，分别表示队列的头部和尾部。   
4.入队操作（`enqueue`）在数组末尾添加元素，出队操作（`dequeue`）从数组头部移除元素。   
5.如果队列满了（`rear` 指向数组末尾的下一个位置等于 `front`），则无法继续入队。   

顺序队列的优点是简单高效，适合于**已知固定大小需求的场景**。缺点是***容量固定***，可能导致空间浪费或队列满时无法继续插入。    

## 链表队列（`Linked Queue`）    
链表队列是使用链表实现的队列，其特点是:       

1.使用***链表***来保存队列中的元素，可以动态分配内存，***无需预先指定固定大小***。   
2.需要两个指针 `front` 和 `rear`，分别表示队列的头部和尾部。   
3.入队操作（`enqueue`）在***链表尾部***__添加节点__，出队操作（`dequeue`）从***链表头部***__移除节点__。   
4.链表队列没有固定容量限制，可以根据需要动态扩展或缩小。   
链表队列的优点是灵活，可以根据实际需求动态调整队列大小，避免了顺序队列固定容量的限制。缺点是相比于顺序队列，可能会有一些额外的空间和时间开销用于管理链表节点。   

## 区别与选择   
### 存储结构：     
__顺序队列__ 使用 __数组__ 存储元素，__链表队列__ 使用 __链表__ 存储元素。   
### 容量限制：   
顺序队列有固定的容量限制，链表队列没有固定大小限制，可以动态扩展。  
### 性能特点：   
顺序队列的入队和出队操作通常比链表队列快，因为***数组的访问速度比链表快。***  
链表队列***可以根据需要动态分配内存，灵活性更高***。  
选择使用顺序队列还是链表队列取决于具体应用场景：   

*如果队列大小固定且已知，且需要高效的入队和出队操作，可以选择顺序队列。     
*如果队列大小不确定或需要动态调整，且对性能要求没有特别高的限制，可以选择链表队列。
